ls -i를 했을 때 나오는 파일 앞 번호는 inode라고 불리는 숫자다.

---

- ln
	- link를 표현
	- 링크의 종류
		- 하드링크와 소프트링크가 있다. 이 둘은 삭제 시 차이가 있다.
		- hardlink(하드링크)
			- ln (하드링크할 파일1 이름) (파일2 이름)
			- 파일1과 파일2는 서로 같은 inode를 공유한다.
			-> 파일을 수정할 경우, 어디서 수정이 이루어지든지간에 수정이 함께 이루어짐.
			- cd 명령어는 inode가 다른 복사본을 만드는 것. 
			- 원본인 파일1을 삭제해도 파일2의 실행에 아무런 문제가 없다.
		- softlink(소프트링크)
			- ln -s (소프트링크할 파일3 이름) (파일4 이름)
			- 윈도우의 바로가기와 유사한 것
			- 파일3과 파일4는 서로 다른 inode를 가진다.
			- 원본인 파일3을 삭제하면 연결이 끊겨 파일4는 무용지물인 파일이 되버린다.
	- 이해를 위한 사이트 링크 : https://reakwon.tistory.com/142
	-> 그림이 수록되어 있어 이해가 편하다.

- touch 
	- 빈 파일 만드는 명령여
	- 해당 파일이 없으면 만들고, 해당 파일이 있으면 수정한다.
	- touch (파일명)	

- grep
	- 검색(파일 내용 검색)
	- grep (검색할 내용) (파일명)

- vi
	- vi 편집기
	- 텍스트 파일을 볼 때 nano 보다는 vi를 더 많이 활용한다.

- find 
	- 파일 검색
	- find /usr/bin -name ls 
		- 파일명이 ls인 파일을 해당 디렉터리 내에서 찾으라는 명령어
	- find /home -user ubuntu 
		- 계정이 ubuntu인 파일을 해당 디렉터리 내에서 찾으라는 명령어
	- find / -user root 	
		- 계정이 root인 파일을 해당 디렉터리에서 찾으라는 명령어(1시간 이상 걸림)
	- 출력 멈추기 : ctrl + c
	- find /tmp -user ubuntu -exec rm {} \; 
		- /tmp 안에서 ubuntu 소유의 모든 파일을 찾는다. 
		- 앞에 찾은 결과물 {} 을 삭제하는 걸 실행하라(-exec). 
			- -exec : execute 실행한다를 의미한다.
			- {} : 집합을 뜻한다.
		- \ : OK를 뜻한다.
