- alias
	- 별칭이란 뜻
	- alias는 사용자가 명령어를 다른 이름으로 바꿔서 사용할 수 있는 쉘 내부 명령어
	- alias
		- 현재 등록된 alias 리스트 확인 가능
	- alias rm='rm -i' 
		- rm이라는 alias를 생성

- unalias
	- alias 삭제 기능
	- unalias rm 
		- 아까 생성했던, rm이라는 alias를 제거

- history
	- 이전에 입력한 명령어들 목록을 보여줌
	- bash shell에서 제공하는 기능이라고 함

- !! 
	- history에 쌓지 않고, 바로 이전의 명령을 실행

- !
	- history 기능
	- !146
		- history에 있는 146번째 입력했던 명령어를 실행
	- !l
		- 맨 마지막에서 시작하여, l로 시작하는 명령어를 찾아 실행

- more ~/.bash_history
	- 자기 홈 root 밑의 bash_ history 파일의 내용 출력

- find
	- 파일 찾기
- grep
	- 문자 찾기

윈도우

시스템 설정 환경 변수

- 시스템 변수 
	- 어떠한 유저가 접속하더라도 먹히는 변수
- 사용자 변수
	- 해당 사용자에서만 먹히는 변수

이 기능이 리눅스 환경 설정에서도 똑같이 먹힌다고 보면 된다고.

하지만 adoptium jdk 를 다운로드받아서 실행하면 adoptium이 자동으로 환경변수에 추가해줌

리눅스 시스템 환경 설정 파일
- 윈도우의 환경 변수와 동일한 개념이라고 생각하도록 하자.

- etc/profile
	- 본셸이나 본셸에 호환되는 모든 셸에 공통으로 적용되는 환경 설정 파일
- etc/bash.bashrc
	- 시스템 공통으로 적용되는 파일
	- 기본 프롬프트를 설정
	- sudo 명령과 관련된 힌트 설정
- etc/profile.d/*.sh
	- 언어나 명령별로 각각 필요한 환경을 설정
	- 필요시 설정 파일 추가
