20240509 리눅스 명령어

---

명령어
- ls 
	- 해당 명령어 뒤에 경로가 올 수 있음(ex)ls /etc/systemd)
	- 해당 명령어 뒤에 파일이 올 수 있음(ex)ls -a) -a : 숨김파일도 다 보여줌
	- 자기 폴더 확인하는 방법 : ls 폴더명. 아무 메세지도 뜨지 않으면 빈 디렉터리이자, 해당 폴더가 있다는 뜻.
	- -R을 추가하면 하위 디렉터리까지 출력한다.(ex)ls -R temp)
- ll
	- 경로 내 파일 목록뿐만 아니라, 파일에 대한 정보까지 다 보여줌( ls -l 에서는 경로 내 파일 목록까지 확인 가능.)
- cd 
	- change directory 약자
	- window에서는 cd c:/ 경로가 리눅스에선 cd /로 표시
	- cd만 입력 시, 해당 계정의 홈 디렉터리로 이동한다.
- . 
	- 현재 디렉터리
- .. 
	- 이전 디렉터리
- touch
	- 파일이 없을 때 : 크기가 0인 파일 생성 
	- 파일이 있을 때 : 해당 파일의 수정날짜 갱신
- mkdir 
	- make directory
	- 디렉터리 생성.
	- 여러 개 동시에 만드는 방법은 스페이스로 디렉터리명을 구분하면 된다.(ex) mkdir a1 a2 a3 )
	- 중간 디렉터리가 있을 때는 -p를 해줘야 한다.(ex) mkdir -p temp/mid/han)
- rmdir
	- remove directory
	- 디렉터리 제거.
	- 중간 디렉터리가 있는 경우에는 한꺼번에 제거를 할 수 없다.
	-> 빈 디렉터리 제거.
- rm
	- 비어있지 않은 모든 디렉터리를 지우기 위한 명령어(ex) rm -rf temp)
	- -rf는 remove false의 약자라고 한다.
- history
	- 그동안 입력한 명령어 내역을 확인할 수 있음.

---

경로 
- 절대 경로 : 기준이 root 
- 상대 경로 : 기준이 자기 자신

리눅스의 경로
- 절대 경로 : /로 시작
- 상대 경로 : /로 시작하지 않음
	- 상대 경로를 제어하는 명령어 : . , ..
		- . : 현재 디렉터리
		- .. : 이전 디렉터리
- ~ : 홈 디렉터리

tmp 폴더에 들어가는 파일은 삭제해도 무방한 파일들이 들어간다.
- 재시작 시 모두 사라지는 파일들이 위치한다.

프로그래밍 시 tmp 폴더를 자주 활용한다고.

---

파일 관련 명령어
- cat
	- 해당 파일을 출력하는 명령어 (ex)cat etc/hosts)
	- -n을 추가하면 줄번호가 추가된다.(ex)cat -n etc/hosts)
- more
	- 화면 단위로 출력
	- space가 화면 단위이다. enter는 행 단위이다.
- less
	- more의 확장판 명령어
	- 파일 내용을 확인하는 명령어인데, 해당 화면을 직접 열어 확인하는 듯이 화면이 넘어간다.
	- 방향키로 확인할 수 있게 해 뒀다.
	- ctrl + B : 한 페이지 뒤로, space : 한 페이지만큼
	- Q : 화면 빠져나오기
	- less로 들어간 화면에서 '/검색어' 형태로 명령어를 입력하고 enter를 누르면 처음으로 찾은 검색어를 보여준다.
	-> 해당 부분에서 n을 누르면 다음 검색어가 있는 위치를 보여준다.
	-> 검색어가 없다면 마지막 페이지를 보여준다. 
- tail
	- 끝에서부터 보여주는 명령어. 
	- cat과 보여주는 방향이 반대.
